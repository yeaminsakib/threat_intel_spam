📄 What Each File Does
Simple explanation of every file's purpose and what goes in and out
⚙️ config.py
The settings file. Every other file imports paths and settings from here.

Think of it like a dictionary of all file paths and important numbers. If you ever want to change where files are saved, you only edit this one file. Nothing is hardcoded anywhere else.

Input: nothingOutput: variables (imported by other files)
🔌 src/email_fetcher.py — Step 1
Reads SpamAssassin corpus files and saves them as .eml files.
Takes in
Files in data/raw/spam/ and data/raw/ham/
Produces
data/raw/spam/collected/*.eml
data/raw/ham/collected/*.eml
data/fetch_summary.json

Key functions: load_corpus_from_folder() — loads one folder, load_all_public_datasets() — runs all folders

🔍 src/header_parser.py — Step 2
Opens every .eml file and pulls out all important header fields into structured data.
Takes in
data/raw/spam/collected/*.eml
data/raw/ham/collected/*.eml
Produces
data/processed/all_emails.json
(one JSON object per email)

Extracts: From, domain, mailserver IPs, Return-Path, DKIM, SPF, hop count, subject, body URLs, and more. This JSON file is the input for everything else.

🗄️ src/ioc_extractor.py — Step 3
Builds YOUR OWN IoC database from the parsed email data.
Takes in
data/processed/all_emails.json
Produces
data/ioc/own_ioc.csv

For each unique IP, domain, URL, and email address found in spam emails — it calculates a threat score and writes one row to the CSV. Ham-only indicators are skipped.

🔗 src/ioc_merger.py — Step 4
Downloads PhishTank and ThreatFox (Abuse.ch) feeds. Merges with your own IoC CSV.
Takes in
data/ioc/own_ioc.csv + internet downloads
Produces
data/ioc/master_ioc.csv
data/ioc/external/phishtank.csv
data/ioc/external/threatfox.csv
🧠 models/train_model.py — Step 5
Trains the machine learning model. Reads emails, builds features, trains, evaluates, saves.
Takes in
data/processed/all_emails.json
Produces
models/spam_classifier.pkl
(prints accuracy, precision, recall, F1)

Uses Random Forest + Logistic Regression ensemble. Features come from header fields: hop count, IP count, DKIM/SPF flags, suspicious TLD, etc.

🤖 src/ai_scorer.py — Step 6
Takes any .eml file and gives it a threat score 0–100 with reasons.
Takes in
Any .eml file + models/spam_classifier.pkl + master_ioc.csv
Produces
JSON: score, decision, reasons list
🔒 src/blocker.py — Step 7
Receives a score result. If score > 75 → blocks IPs/domains. Logs everything.
Takes in
Score result dict from ai_scorer.py
Produces
reports/blocklist.txt
reports/block_log.csv
🚀 pipeline.py — Master Runner
Runs ALL 5 steps above in the correct order with one command.
Step 1: fetch Step 2: parse Step 3: extract IoC Step 4: merge feeds Step 5: train model
python pipeline.py

header_parser.py — the most important function

src/header_parser.py — extract_ips_from_received() Finds attacker IPs
def extract_ips_from_received(received_headers): # Received headers look like this: # "from mail.evil.com ([185.220.101.5]) by ..." # We want to pull out 185.220.101.5 all_text = " ".join(received_headers) # Regex pattern for IPv4 addresses ips = re.findall(r'\b(?:\d{1,3}\.){3}\d{1,3}\b', all_text) # Remove private IPs — they are inside your network # We only care about PUBLIC internet IPs public_ips = [ ip for ip in ips if not (ip.startswith("127.") # localhost or ip.startswith("10.") # private network or ip.startswith("192.168.")) # home/office network ] return list(set(public_ips)) # remove duplicates

ioc_extractor.py — the scoring formula

src/ioc_extractor.py — calculate_threat_score() Scores each IoC 0–100
def calculate_threat_score(spam_count, ham_count, seen_count): # spam_count = how many spam emails contained this IP/domain # ham_count = how many LEGITIMATE emails contained it # seen_count = total emails that had it spam_ratio = spam_count / seen_count # 0.0 to 1.0 # Base score: if seen in 80% spam → base = 64 base = int(spam_ratio * 80) # Bonus for high frequency (seen in many spam emails) if spam_count >= 10: base = min(base + 15, 100) # very confident # Penalty if also seen in legitimate emails # e.g. Google's IP might appear in both spam and ham if ham_count > 0: base = max(base - 20, 0) return base

train_model.py — feature engineering

models/train_model.py — engineer_features() Turns email data into numbers for ML
# ML cannot read text — everything must become a number # Each email becomes one row of numbers (a "feature vector") row = { "received_hops" : 3, # how many servers it passed through "num_mailserver_ips" : 2, # how many public IPs were found "has_dkim" : 1, # 1=yes, 0=no "spf_score" : 3, # 0=pass, 1=none, 2=softfail, 3=fail "spf_failed" : 1, # 1 if SPF failed "domains_match" : 0, # 0 = mismatch = suspicious "num_body_urls" : 12, # many URLs = spam signal "subject_suspicious" : 1, # contains "free", "win", "prize" "suspicious_tld" : 1, # .tk .ml .xyz etc. } # label → 1 = spam, 0 = ham (legitimate)

~8,000+ rows

Combined: own IoC + PhishTank + ThreatFox feeds.

~12,000+ rows

The trained ML model saved to disk.

Random Forest + LR ensemble

Target: 95%+ accuracy. Loaded instantly by ai_scorer.py to score any new email in milliseconds.

Plain text list of blocked IPs and domains. One per line.

Auto-grows as threats detected
